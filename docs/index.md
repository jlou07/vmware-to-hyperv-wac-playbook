# VMware to Hyper-V Migration with Windows Admin Center

> Source article: https://jlou.eu/vmware-hyperv-wac/

This repository is a documentation-first partner playbook based on the source article above.

## Purpose

Following VMware licensing changes, many partners and customers need a practical and controlled VMware exit path. This playbook explains how to migrate virtual machines from VMware to Hyper-V by using the VM Conversion extension in Windows Admin Center (WAC).

## What This Playbook Is

- A presales and delivery-alignment guide for partner teams
- A pragmatic migration workflow focused on low disruption
- A positioning framework for hybrid and Azure-adjacent conversations

## What This Playbook Is Not

- Not an unsupported automation package
- Not a one-click migration tool
- Not a replacement for validation, testing, and rollback planning

## Key Message for Partners

This approach enables a fast and controlled first step away from VMware while preserving optionality for future hybrid strategy.

## Scope

This playbook covers:

- Prerequisites across Windows Admin Center, VMware, and Hyper-V
- A partner-ready migration workflow
- Limitations and delivery caveats
- Hybrid positioning for longer-term roadmap conversations

## Important Support Note

The WAC VM Conversion extension is in Preview. Treat this as a guided migration approach that requires proper lab validation, clear customer communication, and acceptance of Preview constraints.
