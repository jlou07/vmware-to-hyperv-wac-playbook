# Migration Workflow

Use this phased workflow to keep migration activity predictable and aligned with customer risk tolerance.

## 1. Qualify the Opportunity

- Confirm VMware exit drivers and timeline pressure
- Identify workloads suitable for first-wave migration
- Align on acceptable downtime and service windows

## 2. Assess and Prepare

- Validate all prerequisites in the source and target environments
- Build migration waves by business criticality
- Define rollback and communication plans per wave

## 3. Execute Pilot Wave

- Start with low-risk representative VMs
- Perform conversion through WAC VM Conversion
- Validate functionality, performance, and connectivity post-cutover

## 4. Scale by Controlled Waves

- Move from pilot to production waves only after sign-off
- Keep change sizes manageable and reversible
- Track outcomes and adjust sequencing based on observed constraints

## 5. Stabilize and Handover

- Validate operational ownership on Hyper-V
- Capture lessons learned and residual technical debt
- Document customer next steps for hybrid roadmap decisions

## Partner Guidance

- Position this as a controlled migration method, not an instant replatform
- Keep customer messaging clear that VM Conversion in WAC is Preview
- Use pilot evidence to de-risk larger execution waves
